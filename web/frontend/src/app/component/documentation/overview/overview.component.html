Introduction

NCI Enterprise Vocabulary Services (EVS) provides terminology content, tools, and services to NCI and the biomedical research community.

  • For browser support or suggestions, use the Contact Us page.
  • For all EVS resources, services and contacts, see the EVS Web site http://evs.nci.nih.gov/.

EVS Explore Browser provides a consistent, user-friendly tool to browse, search and retrieve content from NCI Thesaurus (NCIt). For information on new features and known issues, see EVS Explore Browser change log.

This help file provides basic information about how to use the EVS Explore Browser effectively, as well as links to additional information elsewhere. The following typeface font conventions are used for describing search and the browser interface:
  • Bold: Browser links, buttons, page tabs, and drop-down boxes.
  • Fixed Width: Search strings.
  • Italics: Concept terms.

Using the Search Box

The Search Box, in the upper left of the EVS Explore Browser welcome page and search results pages and on other pages can be easily found by clicking the Back to Search button. There are a number of controls that can be used to tune searches.

  • Input Box:Enter a string of at least 3 characters you want to search for.
    • Search is not case sensitive (e.g., aids will match aids, Aids, and AIDS).
    • There are no wildcard characters. All characters are matched literally (e.g., searching for Starts With NAT2* will match NAT2*5 Allele but not NAT2 Gene).
    • There is no need to use quotes as they will be ignored by searches
    • Searching can be done with one or multiple words.
    • The search is generally performed against the code, all of the concept names, all of the concept definitions, and all of the property values. The indexing is designed so that anything you would expect to be searchable should produce results.
  • Match type radio buttons determine how your search string (entered into the Input Box) will be matched.
    • Contains is the default type. It performs a search that looks for any or all words typed, favoring results with more words matched, and with words matched in order.
    • Exact Match performs a search that looks for matches having exactly the search words entered in the specified order.
    • Starts With performs a search that looks for matches starting with search words entered in the specified order.
    • Phrase performs a search that looks for matches containing all search words entered in the specified order (but does not require a match at the beginning or an exact match). Searches in this mode will also find all results returned by the same search using Exact Match or Starts With.
    • Fuzzy performs a search that looks for any or all words typed (or fuzzy variants of those words), favoring results with more words matched, and with words matched in order.
    • AND performs a search that looks for matches containing all words entered in any order.
    • OR performs a search that looks for matches containing any (or all) words entered. Searches in this mode will also find all results returned by the same search using ALL.
  • Source dropdown allows searches to be restricted to those concepts having a synonym with "synonym source" matching one of the selected entries in the dropdown. For example, selecting CDISC will only return concepts with a CDISC synonym
  • Search button starts a search.
  • Reset Search button resets all search parameters and returns to the welcome screen.

Search Results

When a search is performed, Search Results are displayed with some basic concept information and a link to load a dedicated view for just that one concept. Clicking this link opens that view in a new window. Other details:

  • All matching concepts are returned.
  • Results are listed from best match to weakest. For example, a Contains search on Bone returns Bone at the top, followed by concepts with two word matches (e.g., Mouse Bone), followed by concepts having less relevance.
  • The total number of search results found along with the search time in milliseconds is shown at the top of the search results table.
  • When a search is performed, it looks for matches among a variety of fields of the concept object, including the preferred name of concept, the code, the other synonyms of the concepts, the definitions of the concept, and the property values of the concept. For this reason, the match may be to something other than the preferred name of the concept.
  • Search results are paged, with a default page size of 10. This can be adjusted with the page size control at the top and bottom of the search results table. The same control will also allow you to navigate to the next page of results (or the previous page where appropriate).
  • The first column of the search results table contains a icon. When clicked, it reveals a panel beneath the corresponding search result that shows highlighted portions of the concept that were matched to the search text provided.
  • By default, the search results table shows the concept code, the preferred name, synonyms, and definitions. The Show Less button can be clicked so only the code and preferred name are shown. In that mode, the Show More button can be clicked to toggle back to the default state. To the right of the Show More button is a control that allows you to select exactly which of the available columns you want shown in the search results display.
  • If a concept is retired, this will be indicated by the phrase "(Retired Concept") next to the name of the concept in the Results listing.
  • A Back to Search button appears in the upper right when you leave the results page to view concept details. Clicking that button will return you to the standard search view.

Concept Details

Detailed information on a selected concept is shown when that concept code is selected from the search results. The concept detail view is loaded into a new tab for display.

The top of this view shows the preferred name and concept code along with an Open in Hierarchy" button. Clicking on that button will open the hierarchy view.

Below that, there are three tabs: Terms and Properties (the default tab), Relationships, and View All. The contents of these tabs are described below.

  • Terms and Properties shows detailed information about the terms and properties of the concept being show. This information is broken down into five tables.
    • The first table shows top level information about the concept, including the preferred name, the display name, the label, and the concept code.
    • Definitions shows any concept definitions (or ALT_DEFINITIONs) in a sortable table. This table includes columns for the definition values themselves, source of the definition, and any associated attribution.
    • Synonyms & Abbreviations shows all of the terms associated with this concept in a sortable table. This table includes columns for the term, the sub-source, the term type, the sub-source code, and the sub-source name.
    • Other Properties shows other properties associated with the concept in a sortable table. This table includes columns for the property type and value. In some cases, property values may be linked out to external data sources that provide additional information about those values. For example, a "CAS_Registry" property will have its value linked out to http://chem.sis.nlm.nih.gov/ to resollve that code.
    • Maps To shows information about maps to other terminologies in a sortable table. This table includes columns for the target name, relationship to target, target term type, target code, and the target terminology (and optionally version).
  • Relationships shows information about related concepts. This information is broken down into seven tables.
    • Parent Concepts shows basic information about parent concepts in a sortable table. This table includes columns for code and preferred name.
    • Child Concepts shows basic information about child concepts in a sortable table. This table includes columns for code and preferred name.
    • Role Relationships shows basic information about asserted or inherited role relationships, pointing from the current concept to other concepts in a sortable table. This table includes columns for code and preferred name.
    • Associations shows basic information about associations pointing from the current concept to other concepts in a sortable table. This table includes columns for code and preferred name.
    • Incoming Role Relationships shows basic information about asserted or inherited role relationships, pointing from other concepts to the current concept in a sortable table. This table includes columns for code and preferred name.
    • Incoming Associations shows basic information about associations pointing from other concepts to the current concept in a sortable table. This table includes columns for code and preferred name.
    • Disjoint With shows basic information about disjointness assertions between this concept and other concepts.
  • View All combines the content of both views listed above into a single view.

Hierarchy

The hierarchy view is enabled by clicking on the Open in Hierarchy button when viewing concept details .

In this view, there is a Concept Hierarchy view along the left side of the display opened to reveal the full path from the root level of the tree to the currently selected concept. The entry for the current concept is highlighted and all sibling concepts and ancestor concepts are also shown in as represented in the hierarchy of the terminology. The page will automatically scroll to the selected concept in this view.

Clicking on the icon to the left of an concept in this view triggers a call to look up and display its child concepts.

Clicking on a concept itself will change the focus of the display to that concept and load the corresponding concept detail information for that concept on the right side of the display.

Metadata Documentation

Along the top toolbar, there is a "Documentation" dropdown that when clicked reveals a number of sub-entries for the various kinds of metadata, including: Associations, Properties, Qualifiers, Roles, Term Types, Sources, Synonym Types, and Definition Types.

Choosing any of these items loads a sortable two or three-column table with the codes, names, and definitions of the metadata elements of the specified type.
